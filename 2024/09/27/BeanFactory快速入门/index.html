BeanFactory快速入门( DI/IOC 思想的实现)

IOC思想的实现：

  1. 导入spring的jar包或者Maven坐标；

  2. 自定义一个service接口以及他的impl实现类；

  3. 创建beans.xml配置文件，将实现类的信息配置到xml中；

  4. 编写测试代码，创建BeanFactory，加载配置文件，获取service实例对象。

    1. 创建工厂对象

    2. 创建读取器（读取xml文件）

    3. 读取配置文件给工厂

    4. 根据id获取Bean实例对象

    DI依赖注入

    1. 自定义UserDao接口以及其impl实现类；

    2. 修改impl代码，新增一个setUserDao（UserDao userDao）用于接收注入对象；

    3. 修改bean.xml配置文件，在UserDaoImpl中的< bean >标签中去嵌入< property >且配置注入;

    4. 修改测试代码，获得service时，setUserService方法执行了注入操作。

ApplicationContext快速入门

简介：

  • ApplicationContext称为Spring容器，内部封装了BeanFactory，比BeanFactory功能更丰富更强大，使用ApplicationContext进行开发时，xml配置文件的名称习惯写成applicationContext.xml

实例：

  1. 创建ApplicationContext，加载配置文件，实例化容器。
  2. ApplicationContext applcationContext = new ClassPathxmlApplicationContext(“applicationContext.xml”)；
  3. 根据beanName获得容器中的Bean实例。
  4. UserService userService = （UserService） applicationContext.getBean(“userService”)；

BeanFactory与ApplicationContext的关系

  • BeanFactory是Spring的早期接口，称为Spring的Bean工厂，ApplicationContext是后期更高级的接口，称之为Spring容器；
  • ApplicationContext在BeanFactory基础上对功能进行了扩展，列如：监听功能，国际化功能（MessageSource）等。BeanFactory的API更偏向底层，ApplicationContext的API大多数都是对这些底层的封装。
  • Bean创建的主要逻辑和功能都被封装在了BeanFactory中，ApplicationContext不仅继承了BeanFactory，而且和ApplicationContext内部还维护着BeanFactory的引用，所以，ApplicationContext与BeanFactory既有继承关系，又有融合关系。
  • Bean初始化时机不同，原始BeanFactory是在首次调用getBean时才进行Bean的创建，而ApplicationContext则是配置文件的加载，容器一创建就将Bean都实例化且初始化好。
